# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please report it responsibly.

**Do not** open a public GitHub issue for security vulnerabilities.

Email: **lopserf@gmail.com**

Include:

- Description of the issue
- Steps to reproduce
- Impact assessment (if known)

We will acknowledge receipt within a few days and work on a fix when appropriate.

## Secrets and credentials

Never commit `.env`, API keys, database passwords, OAuth client secrets, or session tokens to this repository. Use [`.env.example`](.env.example) for placeholders only.

## Supported versions

Security fixes are applied to the latest release on the default branch.
